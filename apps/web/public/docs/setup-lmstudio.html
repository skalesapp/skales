Local AI Setup
🎬

LM Studio + Skales

Desktop app for discovering and running local AI models. Built-in model browser, automatic GPU acceleration, and an OpenAI-compatible server — no command line needed.

← Back to Skales Guide

1 · Download LM Studio

1

Go to lmstudio.ai and download the installer for your OS (Windows, macOS, or Linux).

2

Install and launch LM Studio. The app opens to the Discover tab where you can search for models.

💡 Recommended Models

For a great first experience, search for: Llama-3.1-8B-Instruct, Mistral-7B-Instruct, or Qwen2-7B-Instruct. Choose a Q4_K_M quantization for a balance of speed and quality.

2 · Download a Model

1

In LM Studio, click the Search (🔍) icon in the left sidebar to open the model browser.

2

Search for a model name (e.g. llama-3.1-8b-instruct), select a version, and click Download.

3

Wait for the download to complete. Models are stored in ~/LM Studio/models/.

3 · Start the Local Server

1

Click the Local Server icon (↔) in the left sidebar.

2

Select your downloaded model from the model dropdown at the top.

3

Click Start Server. You'll see a green status: Server running on port 1234.

ℹ️ Default Port

LM Studio's default server port is 1234. You can change this in the server settings if another app is already using that port.

4 · Configure Skales

1

Open Skales → Settings → LLM Provider

2

Select OpenAI Compatible from the provider dropdown.

3

Set the endpoint URL:

http://127.0.0.1:1234/v1

Use 127.0.0.1 instead of localhost to avoid IPv6 resolution issues.

4

Leave the API Key field empty — LM Studio's local server doesn't require authentication.

5

Click Fetch Models. Your loaded model will appear in the dropdown.

6

Tool Calling is ON by default. Select your model and start chatting.

✅ Verification

Send a test message in Skales. If you get a reply, the connection is working. You can also check LM Studio's server log — it shows each incoming request.

Tips & Best Practices

GPU Acceleration LM Studio automatically detects and uses your GPU. Check the server settings to configure GPU layer offloading.
Multiple Models You can have multiple models downloaded. Switch between them in LM Studio's server tab — then click Fetch Models in Skales to update.
Context Size Reduce the context window in LM Studio's server settings for faster responses on limited hardware.
Use 127.0.0.1 Always use 127.0.0.1 in the endpoint URL, not localhost — avoids IPv6 issues on Windows.
Quantization Q4_K_M is the sweet spot. Download Q5_K_M if you have extra RAM. Q2_K if you're on a very constrained system.
Keep Server Running LM Studio's server must be running while you use Skales. It doesn't start automatically — open LM Studio and start the server each session.

Troubleshooting

Problem Fix
Connection refused Start the local server in LM Studio first. Click the ↔ icon → select a model → click Start Server.
Fetch Models returns empty The server is not running, or the port doesn't match. Verify LM Studio shows "Server running on port 1234" and check your Skales endpoint URL.
Slow responses Try a smaller model (3B–7B), or enable more GPU layers in LM Studio's server settings under GPU Offload.
"Ollama not installed" Select OpenAI Compatible in Skales Settings — not the Ollama provider. LM Studio uses the OpenAI-compatible endpoint.
Tool calls failing Ensure the Tool Calling toggle is ON in Skales Settings. Use an instruct-tuned model (not a base model).
Wrong port Check LM Studio's server tab for the actual port in use. Update the endpoint URL in Skales Settings accordingly.
